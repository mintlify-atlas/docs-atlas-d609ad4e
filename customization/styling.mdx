This guide covers general styling customizations beyond colors and fonts, including spacing, layouts, borders, shadows, and component-specific styles.

## Design System Overview

The Cursor UI Clone uses a vanilla CSS approach with:
- **CSS Variables** for theming
- **Utility classes** for common patterns
- **Component-specific styles** for unique elements
- **Consistent spacing system**
- **Flexible grid layouts**

## Spacing System

The design uses consistent spacing values throughout:

### Container Margins

```css
.hero {
    margin: 115px 90px;  /* Vertical: 115px, Horizontal: 90px */
}

.container {
    margin: 0px 90px 60px 90px;  /* Top: 0, Sides: 90px, Bottom: 60px */
}

.trusted .grid {
    margin: 10px 90px 90px 90px;
}
```

**Standard horizontal margin: `90px`**

### Customizing Spacing

Create CSS variables for consistent spacing:

```css
:root {
    --spacing-xs: 4px;
    --spacing-sm: 10px;
    --spacing-md: 20px;
    --spacing-lg: 40px;
    --spacing-xl: 90px;
    --spacing-2xl: 115px;
}
```

Then apply throughout:

```css
.hero {
    margin: var(--spacing-2xl) var(--spacing-xl);
}

.container {
    margin: 0 var(--spacing-xl) 60px var(--spacing-xl);
}
```

<Tip>
Using CSS variables for spacing makes it easy to adjust the entire layout's density with a few value changes.
</Tip>

## Border Radius System

The design uses subtle, consistent border radii:

| Element | Radius | Purpose |
|---------|--------|----------|
| Cards | 4px | Subtle roundness |
| Buttons | 22px, 30px | Pill-shaped CTAs |
| Button List | 8px | UI components |
| Testimonial cards | 4px | Content cards |
| Badge/Tag | 20px | Small UI elements |

```css
    border-radius: var(--radius-full);

```

## Shadows and Depth

The design uses minimal shadows for subtle depth:

```css

    box-shadow: 0 1px 1px rgba(255, 255, 255, .5);
}

/* Controls container - darker shadow */
.controls-container {
    box-shadow: 0 2px 8px rgba(0, 0, 0, 0.3);

/* Buttons explicitly have no shadow */
.nav__right .btn {
    box-shadow: none;
}
```

### Creating a Shadow System

```css
:root {
    --shadow-sm: 0 1px 2px rgba(0, 0, 0, 0.05);
    --shadow-md: 0 2px 8px rgba(0, 0, 0, 0.1);
    --shadow-lg: 0 4px 16px rgba(0, 0, 0, 0.15);
    --shadow-xl: 0 8px 32px rgba(0, 0, 0, 0.2);
    
    /* Light shadows for dark themes */
    --shadow-light: 0 1px 1px rgba(255, 255, 255, 0.5);
}
```

Apply to interactive elements:

```css
.card:hover {
    box-shadow: var(--shadow-lg);
    transition: box-shadow 0.3s ease;
}
```

<Note>
The original design uses minimal shadows, maintaining a flat aesthetic. Add shadows sparingly to preserve the design language.
</Note>


</Tip>

## Transitions and Animations

The design uses minimal animations for smooth interactions:

### Existing Transitions

```css
/* Theme button transitions */
.theme-btn {
    transition: background 0.2s ease;
}

/* Language button transitions */
.language-btn {
    transition: background 0.2s ease;
}
```

### Adding Smooth Transitions

```css
/* Smooth color transitions */
.nav__center a {
    transition: color 0.2s ease;
}

.btn-primary {
    transition: background 0.2s ease, transform 0.1s ease;
}

.btn-primary:hover {
    transform: scale(1.02);
}

/* Card transitions */
.card {
    transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);
}
```

### Subtle Entrance Animations

```css
@keyframes fadeInUp {
    from {
        opacity: 0;
        transform: translateY(20px);
    }
    to {
        opacity: 1;
        transform: translateY(0);
    }
}

.card {
    animation: fadeInUp 0.6s ease forwards;
}

/* Stagger animation for grid items */
.features__grid__card:nth-child(1) { animation-delay: 0.1s; }
.features__grid__card:nth-child(2) { animation-delay: 0.2s; }
.features__grid__card:nth-child(3) { animation-delay: 0.3s; }
```

<Tip>
Keep animations subtle and performant. Use `transform` and `opacity` for best performance as they don't trigger layout recalculation.
</Tip>

## Responsive Design Considerations

While the current styles don't include media queries, here's how to add responsive behavior:

### Breakpoint System

```css
/* Mobile first approach */
:root {
    --breakpoint-sm: 640px;
    --breakpoint-md: 768px;
    --breakpoint-lg: 1024px;
    --breakpoint-xl: 1280px;
}
```

### Responsive Grid Example

```css
/* Mobile: 1 column */
.testimonials__grid {
    grid-template-columns: 1fr;
}

/* Tablet: 2 columns */
@media (min-width: 768px) {
    .testimonials__grid {
        grid-template-columns: repeat(2, 1fr);
    }
}

/* Desktop: 3 columns */
@media (min-width: 1024px) {
    .testimonials__grid {
        grid-template-columns: repeat(3, 1fr);
    }
}
```

### Responsive Spacing

```css
/* Mobile */
.hero {
    margin: 40px 20px;
}

/* Tablet and up */
@media (min-width: 768px) {
    .hero {
        margin: 80px 60px;
    }
}

/* Desktop */
@media (min-width: 1024px) {
    .hero {
        margin: 115px 90px;
    }
}
```

## Advanced Customization

### CSS Custom Properties for Theming

Extend the CSS variable system for complete theme control:

```css
:root {
    /* Colors */
    --color-theme-bg: #14120b;
    --color-theme-fg: #edecec;
    
    /* Spacing */
    --spacing-unit: 10px;
    --container-padding: 90px;
    
    /* Typography */
    --font-primary: 'Gothic', 'Inter', sans-serif;
    --font-ui: system-ui, sans-serif;
    --font-size-base: 16px;
    
    /* Border radius */
    --radius-sm: 4px;
    --radius-md: 8px;
    --radius-lg: 22px;
    --radius-pill: 30px;
    
    /* Transitions */
    --transition-fast: 0.1s;
    --transition-base: 0.2s;
    --transition-slow: 0.3s;
}
```

### Using the Custom Properties

```css
.hero {
    margin: 115px var(--container-padding);
    font-family: var(--font-primary);
}

.btn-primary {
    border-radius: var(--radius-lg);
    transition: background var(--transition-base) ease;
}

.card {
    border-radius: var(--radius-sm);
}
```

### Dark/Light Theme Toggle

```css
:root {
    --color-theme-bg: #14120b;
    --color-theme-fg: #edecec;
}

[data-theme="light"] {
    --color-theme-bg: #ffffff;
    --color-theme-fg: #1a1a1a;
    --color-theme-card-hex: #f5f5f5;
    /* Override all color variables */
}
```

Toggle with JavaScript:

```javascript
document.documentElement.setAttribute('data-theme', 'light');
```

## Best Practices

1. **Use CSS variables** for values that repeat or might change
2. **Maintain consistent spacing** across components
3. **Keep animations subtle** and performance-conscious
4. **Test hover states** for accessibility and visual feedback
5. **Consider responsive behavior** even if not initially implemented
6. **Comment complex CSS** for future maintainability
7. **Group related styles** for easier navigation

<Note>
The vanilla CSS approach makes this project highly portable and easy to understand. Maintain this simplicity when adding customizations.
</Note>
