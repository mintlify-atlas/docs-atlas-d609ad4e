## Overview

The Cursor UI Clone uses a clean, semantic HTML5 structure with a single-page architecture. The entire application is contained in `index.html` with a modular section-based layout that prioritizes accessibility and maintainability.

## Document Structure

### Head Configuration

```html index.html
<head>
  <meta charset="UTF-8" />
  <link rel="icon" type="image/svg+xml" href="/favicon.ico" />
  <link rel="stylesheet" href="styles.css">
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Cursor</title>
</head>
```

**Key Features:**
- UTF-8 character encoding for international character support
- Viewport meta tag for responsive design
- Single external stylesheet (`styles.css`)
- Minimal dependencies - vanilla HTML/CSS only

## Semantic Layout

The page follows a traditional semantic structure with clear separation of concerns:

```
index.html
├── <header>           # Navigation bar
├── <main>             # Primary content container
│   ├── <section>      # Hero section
│   ├── <section>      # Trusted companies
│   ├── <section>      # Feature cards
│   ├── <section>      # Testimonials
│   ├── <section>      # Features grid
│   ├── <section>      # Changelog
│   ├── <section>      # Join us CTA
│   ├── <section>      # Recent highlights
│   └── <section>      # Download CTA
└── <footer>           # Footer navigation and info
```

## Major Sections

### Header & Navigation

<Accordion title="Navigation Structure">

The header uses a sticky position and contains a three-part navigation layout:

```html index.html
<header>

    <!-- Left: Logo and brand -->
    <div class="nav__left flex justify-center align-center">
      <img class="nav__left__logo" src="AVATAR_CIRCLE_2D_DARK.png" alt="Cursor Logo">
      <h2>CURSOR</h2>
    </div>

    <!-- Center: Navigation links -->
    <div class="nav__center">
      <a href="#">Product</a>
      <a href="#">Enterprise</a>
      <a href="#">Pricing</a>
      <a href="#">Resources</a>
    </div>

    <!-- Right: Action buttons -->
    <div class="nav__right">
      <button type="button" class="btn btn-primary">Sign in</button>
      <button type="button" class="btn btn-secondary">Download</button>
    </div>
```

**BEM Naming Convention:**
- Block: `nav`
- Elements: `nav__left`, `nav__center`, `nav__right`, `nav__left__logo`
- Modifiers: `btn-primary`, `btn-secondary`

  <Tab title="Layout Behavior">

**Alternating Layout:**
- Cards use `flex-direction: row` for odd cards (content left, image right)
- Cards use `flex-direction: row-reverse` for even cards (image left, content right)
- This creates visual variety while maintaining code consistency

**CSS Implementation:**
```css styles.css
.card:nth-child(odd) {
    flex-direction: row;
}

.card:nth-child(even) {
    flex-direction: row-reverse;
}
```

## Accessibility Features

<Note>
**Accessibility Considerations:**
- Semantic HTML5 elements (`<header>`, `<nav>`, `<main>`, `<footer>`, `<section>`)
- Proper heading hierarchy (h1 → h2 → h3 → h4)
- `alt` attributes on images (though some are empty and should be improved)
- `<figcaption>` for testimonial attributions
- Button `type` attributes specified
- Keyboard-accessible interactive elements
</Note>

## Key Patterns

### BEM Methodology

The project follows Block Element Modifier (BEM) naming:

```
Block:    .nav, .card, .hero
Element:  .nav__left, .card__item1, .hero__cta
Modifier: .btn-primary, .btn-secondary
```

### Section Container Pattern

Each major section follows this pattern:

1. **Wrapper:** `<section>` with semantic class name
2. **Spacing:** `container` and `p-20` utility classes
3. **Content:** Heading + content grid/flex layout
4. **Imagery:** Images placed in dedicated wrappers

## Performance Considerations

<Accordion title="Asset Loading Strategy">

**Inline vs. External:**
- CSS: External file (`styles.css`) - allows browser caching
- Images: Referenced via `src` attributes - can be lazy loaded
- SVG Icons: Inline in HTML for theme switcher - eliminates HTTP requests
- Fonts: Google Fonts and local font files

**Optimization Opportunities:**
- Add `loading="lazy"` to below-the-fold images
- Consider WebP format for hero images
- Add width/height attributes to prevent layout shift

</Accordion>
