## Overview

The Cursor UI Clone uses a single-file CSS architecture (`styles.css`) with a hybrid approach combining utility classes, component styles, and custom properties. The system is built on vanilla CSS with no preprocessors or frameworks.

## File Structure

```
source/
├── styles.css          # Main stylesheet (681 lines)
├── fonts/
│   └── Gothic.woff2    # Custom font file
└── index.html
```

**Philosophy:**
- Single CSS file for simplicity
- No build process required
- Component-based organization
- CSS custom properties for theming

## Architecture

The CSS file is organized in the following order:

```css
1. Imports (Google Fonts)
2. @font-face declarations
3. :root CSS variables (design tokens)
4. Reset/Base styles
5. Utility classes
6. Layout components
7. Section-specific styles
8. Interactive elements
```

## Imports & Font Loading

<Tabs>
  <Tab title="Google Fonts">

```css styles.css
@import url('https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700&display=swap');
```

**Configuration:** Font family Inter with weights 300, 400, 500, 600, 700 and display strategy `swap` (shows fallback font while loading).

  </Tab>
  <Tab title="Custom Font">

```css styles.css
@font-face {
    font-family: "Gothic";
    src: url("./fonts/Gothic.woff2") format("woff2");
    font-weight: normal;
    font-style: normal;
    font-display: swap;
}
```

**Primary Font Stack:**
```css
font-family: 'Gothic', 'Inter', sans-serif;
```

Gothic is the primary brand font, with Inter as fallback.

## CSS Methodology

### 1. BEM Naming Convention

The project follows BEM (Block Element Modifier) for component classes:

<CodeGroup>

```css Block
/* Block: standalone component */
.nav { ... }
.card { ... }
.hero { ... }
```

```css Element
/* Element: part of a block */
.nav__left { ... }
.nav__center { ... }
.card__item1 { ... }
.card__item2 { ... }
```

```css Modifier
/* Modifier: variant of block/element */
.btn-primary { ... }
.btn-secondary { ... }
.color-theme-card-01-hex { ... }
```

</CodeGroup>

### 2. Utility-First Pattern

Minimal set of utility classes for common layouts:

```css styles.css
.flex {
    display: flex;
}

.justify-center {
    justify-content: center;
}

.justify-between {
    justify-content: space-between;
}

.align-center {
    align-items: center;
}

.p-20 {
    padding: 20px;
}
```

**Usage Example:**
```html
<nav class="nav flex justify-between align-center">
  <!-- Flexbox with space-between and centered items -->
</nav>
```

## Advanced CSS Features

### Color-Mix Function

```css styles.css
:root {
    --color-soft: color-mix(in oklab, var(--color-theme-fg)60%, transparent);
}

.card__item1 h4 {
    color: color-mix(in oklab, var(--color-theme-fg) 60%, transparent);
}
```

<Note>
**Modern CSS Feature:** `color-mix()` allows blending colors in different color spaces. The `oklab` color space provides perceptually uniform mixing, creating more natural-looking transparent colors.
</Note>

## Typography System

```css styles.css
/* Headings */
.hero h1 {
    font-size: 26px;
    font-weight: 400;
}

.testimonials h2 {
    font-size: 2.25rem;  /* 36px */
    font-weight: 400;
}

.features h3 {
    font-size: 1.625rem;  /* 26px */
    font-weight: 400;
}

.trusted h4 {
    font-size: .875rem;   /* 14px */
    font-weight: 200;
}

/* Body text */
.nav__center a {
    font-size: 14px;
}

.card__item1 h3 {
    font-size: 22px;
    font-weight: lighter;
}
```

**Scale:**
- Display: 72px (4.5rem) - Download CTA
- H1: 26px - Hero
- H2: 36px (2.25rem) - Section headers
- H3: 26px (1.625rem) - Subsections
- Body: 14-16px - Most content

**Weight:**
- Light (200-300) for display text
- Normal (400) for most content
- `font-weight: lighter` for subtle hierarchy

## Performance Optimizations

<Note>
**CSS Performance Features:**
- Single file (reduces HTTP requests)
- No preprocessor compilation needed
- Minimal specificity conflicts
- Efficient selectors (mostly class-based)
- Font display: swap (prevents FOIT)
</Note>

## Best Practices

✅ **Strengths:**
- Consistent naming with BEM
- Reusable utility classes
- CSS custom properties for theming
- Modern CSS features (Grid, Flexbox, color-mix)
- No !important declarations
- Semantic class names

<Warning>
**Areas for Improvement:**
- Consider responsive breakpoints (currently no media queries)
- Add CSS comments for section organization
- Extract more repeated values to variables
- Consider CSS custom properties for spacing scale
- Add print stylesheet
</Warning>
