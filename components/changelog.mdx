## Overview

The changelog section displays recent product updates in a compact card grid. Each card shows the release date and a brief description of what was added or improved.

## Visual Description

This section features:
- A section heading: "Changelog"
- Four cards arranged horizontally
- Each card displays a date and update description
- Version numbers are highlighted with rounded badge styling
- A "See what's new" link below the grid

## Version Badge Component

The version number badge is an inline element that can be conditionally included:

```html
<!-- Major release with version -->
<p><span>2.4</span> Jan 22, 2026</p>

<!-- Minor update without version -->
<p>Jan 16, 2026</p>
```

<Warning>
Ensure consistent date formatting across all changelog entries for professional appearance.
</Warning>

## Design Patterns

### Version Badge Styling

| Property | Value | Purpose |
|----------|-------|----------|
| `border` | 1px solid white (65% opacity) | Subtle outline |
| `border-radius` | 20px | Pill shape |
| `padding` | 3px 10px | Compact spacing |
| `margin-right` | 8px | Space before date |
| `font-size` | 0.9rem | Slightly smaller than body |

### Spacing Hierarchy

<Accordion title="Understanding the spacing">
- Section margin-top: 180px (major section separation)
- Grid gap: 10px (tight card spacing)
- Card internal gap: 10px (date to description)
- Section gap: 20px (heading to grid to link)
</Accordion>

## Content Guidelines

### Date Format

The changelog uses this date format:
- Month abbreviated (Jan, Feb, Dec)
- Day with comma (22,)
- Full year (2026)

Example: `Jan 22, 2026`

### Description Length

Keep descriptions concise (3-7 words) to fit within the fixed card height:

<CardGroup cols={2}>
  <Card title="Good" icon="check">
    "Subagents, Skills, and Image Generation"
  </Card>
  <Card title="Too Long" icon="xmark">
    "Introducing new subagent capabilities, enhanced skill management, and AI-powered image generation features"
  </Card>
</CardGroup>

## Responsive Considerations

The four-column grid works well on desktop. For mobile, consider:

```css
@media (max-width: 1024px) {
  .changelog__grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

@media (max-width: 640px) {
  .changelog__grid {
    grid-template-columns: 1fr;
  }
}
```

<Note>
These responsive rules are not in the original source but would improve mobile experience.
</Note>
